# grhsim_opt 文档管理规则

本文是 `pdocs/grhsim_opt` 的独立规则说明文档，用于约束本目录后续的增量记录方式。

## 1. 适用范围

- 本目录中的“记录类文档”统一采用增量归档方式管理。
- 本目录中的“管理类文档”仅包含 `README.md` 与 `RULES.md`，不占用记录类文档的 `NOxxxx` 编号。

## 2. 编号与命名规则

- 记录类文档文件名统一采用 `NOxxxx_主题_YYYYMMDD.md`。
- `NOxxxx` 为四位数字，按记录形成时间单调递增，编号一经分配不得复用。
- 同一天内新增多篇文档时，仍按形成先后继续递增，不按主题分段编号。
- 新文档默认从当前目录已分配的下一个编号继续；当前下一个可用编号为 `NO0026`。

## 3. 增量记录规则

- 本目录采用“只追加、不覆盖”的记录方式。
- 已有文档允许扩展，但只能追加新的章节、补充说明、勘误说明或增量更新记录。
- 不允许删除历史结论、整体替换旧内容、复用旧文件承载新的独立主题。
- 如果新增内容已经构成独立议题，优先新建一篇新的 `NOxxxx` 文档，而不是覆盖旧文档。
- 如果必须修正文档中的错误，应保留原结论上下文，并以“勘误”或“增量更新 YYYY-MM-DD”形式追加说明。

## 4. 链接规则

- 目录内文档互相引用时，统一使用相对路径，例如 `./NO0008_xxx_20260419.md`。
- 跨目录引用时，统一使用相对路径，例如 `../other_dir/doc.md`。
- 不再使用仓库绝对路径或机器本地绝对路径作为 Markdown 链接目标。
- 普通代码块、命令示例、实验产物路径可以保留原始绝对路径文本，但它们不应写成 Markdown 绝对链接。

## 5. 当前历史文档整理规则

- 本目录现有 7 篇历史记录已在本次整理中统一重命名为 `NOxxxx_*.md`。
- 重命名后的稳定标识以文件名中的 `NOxxxx` 为准，文档标题与 `README.md` 索引必须保持一致。
- 后续新增文档直接使用 `NOxxxx_主题_YYYYMMDD.md` 命名，不再新增无编号文件名。

## 6. 时间顺序判定规则

- 优先按文档明确记录的日期排序。
- 如果多篇文档日期相同且没有更细粒度时间戳，则按“依赖关系优先、阅读顺序优先”的原则排序：
  - 基线文档排在对齐/汇总文档之前。
  - 单边机制记录排在双边对照快照之前。
  - 被引用文档排在引用它的文档之前。
- 如果仍然无法区分，则保持当前目录索引中既有顺序，不反复改号。

## 7. 目录维护要求

- 每次新增记录类文档后，必须同步更新 `README.md` 中的时间顺序索引。
- 每次新增目录内交叉引用后，必须检查链接是否仍为相对路径。
- 若旧文档追加了重要增量更新，可在 `README.md` 的备注列中补充“已追加更新”的说明，但不改变其原始编号。
